Girar la manilla hacia la iquierda. 5. Empujar la puerta. 6. Soltar la manilla. 7. Caminar hacia adelante. 8. Darse la vuelta en 180°. 9. Levantar nuevamente la mano. 10. Tomar la manilla. 11. Empujar la puerta suavemente hasta que tope. ¿Que puede hacer un computador? - Sumar. - Restar. - Multiplicar. - Dividir. - Recordar informacion. - Recordar acciones. ¿Que se utiliza para indicar a un computador lo que debe hacer? Estas acciones se realizan mediante un lenguaje de programacion. ¿Que es un lenguaje de programacion? - Es un lenguaje formal. - Con reglas gramaticales definidas. - Nos permite entregarle instrucciones a una maquina. Paradigma: Es como una forma de ver la vida, una forma de como podemos resolver problemas. En este caso el paradigma de la programacion es como una filosofia en cuanto a como se debe resolver algo. OBS: un paradigma en muchas ocasiones es sencillamente una preferencia en cuanto a como resolver algo y nada mas. (no indica que algo este bien o mal hecho). Orientacion a objetos: La Programación Orientada a Objetos (POO) es un paradigma de programación, es decir, un modelo o un estilo de programación que nos da unas guías sobre cómo trabajar con él. Se basa en el concepto de clases y objetos. Este tipo de programación se utiliza para estructurar un programa de software en piezas simples y reutilizables de planos de código (clases) para crear instancias individuales de objetos. Clase: Una clase es una plantilla. Define de manera genérica cómo van a ser los objetos de un determinado tipo. ejemplo: Una clase para representar a animales puede llamarse ‘animal’ y tener una serie de atributos, como ‘nombre’ o ‘edad’ (que normalmente son propiedades), y una serie con los comportamientos que estos pueden tener, como caminar o comer, y que a su vez se implementan como métodos de la clase (funciones). Objeto: Un objeto es el resultado de la instanciación de una clase. ejemplo2: class Personas { edad: Integer nombre: String colorCabello: String decirNombre(): habla {tu nombre} } personaChanchito = new Personas( edad: 36 nombre: "chanchito" colorCabello: "castaño" ) personaFelipe = new Personas( edad: 15 nombre: "felipe" coloCabello: "rojo pasion" ) Metodo: Es como una funcion pero ejecutada dentro del objeto. Condicion: Control de flujo de una aplicacion. Dependiento el dato que estemos viendo se pueden tomar distintas desiciones y ejecutar distintas porciones de codigo/algoritmo. Ejemplo: function entrarALaCasa(persona): si persona.edad > 17: correr hacia la ventana saltar hacerse bolita si no: vete de aqui Errores: function entrarALaCasa(persona): acercarse a la puerta caminar hacia adelante Si la puerta llega a estar cerrada la persona se pegara en la cara y no podra entrar por lo que deberia arrojarse un error. Pero por el contrario si la puerta estubiese abierta la funcion se ejecutaria sin ningun problema. Por lo que en algunos casos nuestro codigo funcionaria y en otros casos no. A eso le denominamos BUG. Bug: Son errores que uno puede introducir de manera involuntaria en el codigo. ejemplo: Si la puerta esta cerrada arrojara error: error ¡puerta cerrada! OBS: dependiendo el lenguaje de programacion que se este utilizando los mensajes de error van a variar y tambien de que es lo que se esta ejecutando. Por eso es importante aprender a leer los mensajes de error que te entrega el lenguaje de programacion con el que estas trabajando. Para asi poder intentar solucionar la logica que tu escribiste en tu codigo, en este caso tu algoritmo.
