¿Que es programar? El programar es un set de intrucciones que uno le da a una maquina. Ejemplo: Entrar a la casa: 1. Acercarse a la puerta. 2. Levantar la mano. 3. Tomar la manilla. 4. Girar la manilla hacia la iquierda. 5. Empujar la puerta. 6. Soltar la manilla. 7. Caminar hacia adelante. 8. Darse la vuelta en 180°. 9. Levantar nuevamente la mano. 10. Tomar la manilla. 11. Empujar la puerta suavemente hasta que tope. ¿Que puede hacer un computador? - Sumar. - Restar. - Multiplicar. - Dividir. - Recordar informacion. - Recordar acciones. ¿Que se utiliza para indicar a un computador lo que debe hacer? Estas acciones se realizan mediante un lenguaje de programacion. ¿Que es un lenguaje de programacion? - Es un lenguaje formal. (Orientado a las computadoras) - Con reglas gramaticales definidas. - Nos permite entregarle instrucciones a una maquina. Dato e Informacion: Un dato es la representación de una variable que puede ser cuantitativa o cualitativa que indica un valor que se le asigna a las cosas y se representa a través de una secuencia de símbolos, números o letras. Como información denominamos al conjunto de datos, ya procesados y ordenados para su comprensión, que aportan nuevos conocimientos a un individuo o sistema. ejemplo1: Dato: 36 chanchito años feliz Informacion: Usuario: 36 años chanchito feliz Ejemplo2: Dato: Salchicha Informacion: perro salchicha Variable: Es un espacio o posicion de memoria que almacena un dato. ejemplo: edad = 10 cantidad = 10 x = "chanchito" animal = "chanchito" menu = "chanchito" holamundo="chanchito feliz" Tipos de datos: animal = "chanchito" -> String (Se refiere a un Texto) edad = 36 -> Integer (Se refiere a un Numero Entero) harina = 0.5 -> Float (Se refiere a un Numero Decimal) caracter = "a" -> Char (Se refiere a un Solo Caracter) casado = true -> Boolean (Solo tiene 2 valores: "True" or "False") Funciones: Permite reutilizar algoritmos que hallamos construido. Son una forma de recordar el algoritmo que escribimos para no tener que repetirselo constantemente a la maquina. Paradigma: Es como una forma de ver la vida, en este caso en como podemos resolver problemas. Debemos pensar en el paradigma de la programacion como una filosofia en cuanto a como se debe resolver algo. OBS: un paradigma en muchas ocasiones es sencillamente una preferencia en cuanto a como resolver algo y nada mas. (no indica que algo este bien o mal hecho). Orientacion a objetos: La Programación Orientada a Objetos (POO) es un paradigma de programación, es decir, un modelo o un estilo de programación que nos da unas guías sobre cómo trabajar con él. Se basa en el concepto de clases y objetos. Este tipo de programación se utiliza para estructurar un programa de software en piezas simples y reutilizables de planos de código (clases) para crear instancias individuales de objetos. Clase: Si lo queremos definir en una sola frase, una clase es un diseño que se puede utilizar para crear varios objetos individuales. Una clase define un grupo o conjunto de datos, en objetos se llaman atributos, que definen los objetos, así como un conjunto de comportamientos, las funciones o métodos del objeto, que lo manipulan y relacionan los objetos unos de otros. Objeto: Un objeto es el resultado de la instanciación de una clase. ejemplo: class Personas { edad: Integer nombre: String colorCabello: String decirNombre(): habla {tu nombre} } personaChanchito = new Personas( edad: 36 nombre: "chanchito" colorCabello: "castaño" ) personaFelipe = new Personas( edad: 15 nombre: "felipe" coloCabello: "rojo pasion" ) Metodo: Un método es una subrutina cuyo código es definido en una clase y puede pertenecer tanto a una clase, como es el caso de los métodos de clase o estáticos, como a un objeto, como es el caso de los métodos de instancia. ejemplo: decirNombre(): habla {tu nombre} dentro de la clase Personas: class Personas { edad: Integer nombre: String colorCabello: String decirNombre(): habla {tu nombre} } Condicion: Control de flujo de una aplicacion. Dependiento el dato que estemos viendo se pueden tomar distintas desiciones y ejecutar distintas porciones de codigo/algoritmo. Ejemplo: function entrarALaCasa(persona): si persona.edad > 17: correr hacia la ventana saltar hacerse bolita si no: vete de aqui Errores: Ejemplo: function entrarALaCasa(persona): acercarse a la puerta caminar hacia adelante Si la puerta llega a estar cerrada la persona se pegara en la cara y no podra entrar por lo que deberia arrojarse un error. Pero por el contrario si la puerta estubiese abierta la funcion se ejecutaria sin ningun problema. Por lo que en algunos casos nuestro codigo funcionaria y en otros casos no. A eso le denominamos BUG. Bug: Son errores que uno puede introducir de manera involuntaria en el codigo. ejemplo: Si la puerta esta cerrada arrojara error: error ¡puerta cerrada! OBS: dependiendo el lenguaje de programacion que se este utilizando los mensajes de error van a variar y tambien de que es lo que se esta ejecutando. Por eso es importante aprender a leer los mensajes de error que te entrega el lenguaje de programacion con el que estas trabajando. Para asi poder intentar solucionar la logica que tu escribiste en tu codigo, en este caso tu algoritmo.
